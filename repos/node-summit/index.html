The Conference

  • It was held in Downtown San Francisco from July 26th to 28th.
  • The first day contained workshops from sponsors.
  • The last two days contained panel discussions, talks, and a couple sponsor workshops.

Demographics

  • Seemed to be around 400 attendees.
  • At least 20% were women.
  • Age distributions were from 20s to 50s.
  • At least 50% were from Bay Area.
  • Quite a few were from Seattle and Portland.
  • Few were from Europe, and they were speakers or sponsors.

Session Notes

Node @ NASA

Collin Estes

  • NASA is using Node.js to build some of the internal services.
  • For security reason, only node.js core modules are used, modules from npm are prohibited.
  • Use proxy to manage Legacy API migration.
  • To introduce new technology, prototype and demo to show value.

Securing Node.js Applications

Panel Discussion

  • npm is really popular and it's getting attacked quite often.
  • You're responsible for what you use.
  • Input validation are extremely important.
  • express.js has lots of security holes, and it's less secured than hapi.js.
  • Package install scripts vulnerability
  • Defeating the npm Worm
  • Node.js Security Checklist
  • Express Production Best Practices
  • node security platform command-line tool - nsp

Addressing the Security Risk in npm Dependencies

Guy Podjarny (snyk)

  • He showed how easy it was to hack a node.js app.
  • Hack to root director from URL with curl.
  • Inject script through link.
  • Use regular expression to cause long run string search task, which blocks the node.js thread.
  • If the code takes in user input to create buffer, it can be hacked by providing a number.
  • Here is the same talk from Fluent Conference.
  • snyk has a test page that you can test and learn about vulnerabilities.

Node.js Monitoring, Performance, and Security

Dan Shaw (NodeSource)

  • Put nginx in front of node app to manage cache, security, and rate limit.
  • Pass in only the data the function needs, don't pass in the whole object. Otherwise, memory will keep growing until the whole function is done.

The Cost of Logging

Matteo Collina & David Mark Clements (nearForm)

  • Logging impacts performance.
  • Disable logging increases throughput.
  • Calling one log function ends up with calling multiple functions.
  • pino is the fastest logger.
  • There were quite a few performance tricks mentioned in the talk.
  • Here is the slides.

Microservices with gRPC, Docker, and Kubernetes

Sandeep Dinesh (Google)

  • Kubernetes contains everyting to build microservices:
    • load balancing
    • service discovery
    • zero downtime deployment
    • no need to SSH to container, use the API to do everything
  • GRPC allows services to communicate with high performance:
    • Protocol Buffers
    • HTTP/2: bidirectional streaming, flow control, header compression, multiplexing requests over a single TCP connection

Building PayPal Into Cross-Domain Web Components

Daniel Brain (PayPal)

  • post-robot: makes postMessaging between windows, popups and iframes a lot more deterministic.
  • beaver-logger: lets you do logging on the client side without worrying about sending hundreds of requests to your server.
  • xcomponent: builds cross-domain components, in iframes and popup windows.

Building Your DevOps for Node.js

Chetan Desai (Intuit)

  • Avoid global dependecy and use local dependency for build isolation.
  • Run npm shrinkwrap to lock down dependencies for production use.
  • Run npm prune --production to reduce the size of node_modules.
  • Never run npm at deployment time.
